## 필수 요구사항 (각 사용자는 PR에 해당 내용을 넣어주세요.)

### 1. 메뉴판 화면 (`/`)

- [x] 메뉴 카테고리를 `GET /api/catalog/categories`로 가져와 탭/버튼으로 표시
- [x] 메뉴 목록을 `GET /api/catalog/items`로 가져와 카드/리스트 형태로 표시
- [x] 카테고리 클릭 시 해당하는 메뉴 목록만 화면에 노출
- [x] 각 메뉴에 `iconImg`을 사용한 이미지, 이름, 가격 표시
- [x] 메뉴 클릭 시 주문하기 페이지(`/menu/:itemId`)로 이동
- [x] CTA(하단 버튼)에 장바구니에 담긴 전체 메뉴의 개수와 가격 표시
- [x] 장바구니가 비어있는 경우 "장바구니 보기" 만 출력

### 2. 주문하기 화면 (`/menu/:itemId`)

- [x] `GET /api/catalog/items/:itemId`로 메뉴 상세 정보 가져오기
- [x] `GET /api/catalog/options`로 옵션 정보 가져오기
- [x] 해당 메뉴의 `optionIds`에 해당하는 옵션만 표시
- [x] 옵션 노출은 서버에서 내려온 순서대로 표시
- [x] **`grid` 타입**: 그리드 형태로 아이콘과 함께 노출, **필수 선택**, 하나만 선택 가능
- [x] **`select` 타입**: 셀렉트박스로 노출, 클릭 시 바텀시트 출력, 바텀시트에서 항목 선택 시 값 업데이트, **선택 사항**
- [x] **`list` 타입**: 체크리스트로 노출, `minCount` ~ `maxCount` 범위 내에서 복수 선택 가능
- [x] `list`에서 최소 선택 개수 미만으로 선택하고 CTA 클릭 시 Toast 등으로 안내 (예: "온도를 선택해주세요")
- [x] `list`에서 최대 선택 개수 초과 시도 시 Toast 등으로 안내 (예: "최대 N개까지 선택할 수 있어요")
- [x] 수량 선택 (1~99)
- [x] 선택한 옵션에 따른 실시간 가격 계산 표시 (CTA에 `수량 * 가격` 노출)
- [x] CTA 클릭 시 장바구니에 추가 후 메뉴판 화면으로 이동
- [x] 존재하지 않는 메뉴 ID 접근 시 적절한 처리 (404)

### 3. 장바구니 화면 (`/cart`)

- [x] 장바구니에 담은 메뉴 이름, 옵션 목록, 가격 노출
- [x] 같은 메뉴 + 같은 옵션끼리 묶어서 수량 표기 (예: 아메리카노 HOT 2잔, ICE 1잔)
- [x] 담은 수량을 반영한 총 가격 표기 (예: 아메리카노 2잔 = 4,500 \* 2 = 9,000원)
- [x] `x` 버튼으로 해당 메뉴-옵션 삭제
- [x] 수량 변경 가능 (증가/감소)
- [x] 장바구니가 비었을 때 빈 상태 안내
- [x] CTA에 전체 주문 개수와 금액 노출
- [x] CTA 클릭 시 `POST /api/orders` API를 호출하여 주문 생성
- [x] 주문 API 실행 중 버튼이 중복 클릭되지 않도록 처리
- [x] `POST /api/orders`에서 에러 응답 시 Toast로 에러 메시지 안내
- [x] 주문 성공 시 장바구니 비우기 + 주문 완료 페이지(`/orders/:orderId`)로 이동

### 4. 주문 완료 화면 (`/orders/:orderId`)

- [x] `GET /api/orders/:orderId`로 주문 정보 가져오기
- [x] 총 개수와 금액을 화면에 노출
- [x] CTA 버튼 클릭 시 메뉴판 화면으로 이동
- [x] 존재하지 않는 주문 ID 접근 시 적절한 처리 (404)

### 상태 처리

- [x] 로딩 중 표시
