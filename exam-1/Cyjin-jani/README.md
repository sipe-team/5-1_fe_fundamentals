# Exam 1: 다중 필터 상품 목록 구현

## 문제 이해

- 이커머스 서비스의 **상품 목록 페이지** 구현.
- 상품의 필터링, 정렬, 검색 기능 (auto complete)를 포함
- API 요청에 대한 응답 처리 시 로딩, 에러 처리 필요.

## 사용한 기술

- **TypeScript** : 필수 조건
- **React** : 필수 조건
- **TailwindCSS** : 가장 익숙한 스타일링 도구. 평가대상에서 제외라서 자유롭게 선택.
- **TanstackQuery** : 서버 상태 관리를 위해 도입. 커스텀 훅을 사용하여 컴포넌트의 역할과 책임 분리.
- **ReactErrorBoundary** : 선언적으로 로딩 상태, 에러 상태 등을 관리하기 위해 도입

## 필수 요구사항

### 상품 목록
- [X] API에서 상품 데이터를 불러와 카드 형태로 렌더링 (이미지, 상품명, 가격, 카테고리, 평점)
- [X] 상품명이 길 경우에도 레이아웃이 깨지지 않도록 처리

### 필터링
- [X] **카테고리 필터**: 체크박스 다중 선택 (shoes, tops, bottoms, accessories)
- [X] **키워드 검색**: 상품명 검색 (디바운싱 적용)

### 검색 자동완성
- [ ] 검색 input에 글자를 입력하면 자동완성 API를 호출하여 **드롭다운으로 제안 목록 표시**
- [ ] 제안 항목 클릭 시 해당 검색어로 검색 실행
- [ ] 적절한 시점에 드롭다운 닫기 (선택, 외부 클릭, Esc 등)

### 정렬
- [ ] 가격 낮은순 / 가격 높은순 / 최신순 / 평점순

### URL 동기화
- [X] 필터/정렬/검색 상태가 URL 쿼리스트링에 반영
- [X] 새로고침해도 필터 상태 유지
- [X] URL을 복사해서 공유하면 같은 필터 상태로 열림

### 필터 초기화
- [ ] "전체 초기화" 버튼으로 모든 필터를 기본값으로 리셋

### 상태 처리
- [X] 로딩 중 표시
- [X] API 에러 시 사용자 안내 + 재시도 가능
- [ ] 필터 결과가 없을 때 빈 상태 안내

## 선택 요구사항 (보너스)

- [ ] UI/UX 개선 (사용자가 실제로 이 서비스를 사용했을 때를 고려하여, UI/UX 에 대해 어필할 포인트가 있다면 어필해주세요!)
- [ ] 자동완성에서 키보드 탐색 지원 (↑/↓ 화살표로 항목 이동, Enter로 선택)
- [ ] 자동완성에서 입력 중인 글자와 매칭되는 부분 하이라이트
- [ ] 필터 결과 개수 표시 (예: "32개 상품")
- [ ] 반응형 레이아웃 (모바일 / 데스크톱)
- [ ] 상품 목록 애니메이션 / 트랜지션
- [ ] 페이지네이션 또는 무한 스크롤

## PR 작성 가이드

PR 본문에 아래 항목들을 포함해주세요. 코드만큼이나 **설계 판단의 근거**가 중요합니다.

### 1. 필터링 전략
- 서버 사이드 필터링
- 선택 이유
    - 데이터 수가 아주 많지 않은 경우에 적합
    - 필터 시마다 서버 데이터를 조회하므로 최신 데이터를 보여주는 데 유리함
      - 미리 받아서 보여주는 경우에, 일정 간격마다 data를 갱신해주지 않으면 최신 데이터를 바로 보기 어려울 수 있음 
    - 반면에, 데이터가 자주 변경되지 않는 데이터의 경우라면 필터링 마다 최신 정보를 불러오지 않아도 되므로 클라이언트 사이드가 더 유리할 수 있음.
      - 서버 요청 횟수가 줄고, 비용 감소

### 2. 상태 관리 전략
> 필터 상태를 어디서 관리하나요? (URL 파라미터, useState, 전역 store 등)
> 그렇게 선택한 이유는?

### 3. 컴포넌트 구조
> 어떤 기준으로 컴포넌트를 나눴나요?
> 폴더 구조는 어떻게 잡았고, 왜 그렇게 했나요?

### 4. 검색 및 자동완성 구현
> 디바운싱을 어디에(어느 레이어에) 걸었나요? 시간은 얼마로 설정했고, 왜?
> 자동완성 드롭다운의 열기/닫기는 어떻게 관리했나요?
> 키보드 접근성은 어떻게 처리했나요? (선택사항이지만 시도했다면 설명해주세요)

### 5. 에러/재시도 처리
> API가 간헐적으로 실패하는 상황에서 어떤 UX를 제공했나요?
> 재시도 로직은 어떻게 구현했나요?

### 6. 과제를 수행하는데 어려움은 없었나요?
> 어려웠던 부분이 있다면 말씀해주세요~! 추후 과제때 개선해보겠습니다.

### 7. 추가 라이브러리 (optional)
> 추가로 설치한 라이브러리가 있다면, 각각 왜 선택했는지 설명해주세요.

### 8. 아쉬운 점 / 개선하고 싶은 점 (optional)
> 시간이 더 있었다면 어떤 부분을 개선했을지 자유롭게 적어주세요.
