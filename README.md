## 🔧 참여 방법

### 1. Clone

```bash
git clone https://github.com/{organization}/5-fe-fundamentals.git
cd 5-fe-fundamentals
```

### 2. 브랜치 생성 후 본인 폴더에서 개발

```bash
# 브랜치 생성
git checkout -b exam-1/{github-username}

# 본인 폴더로 이동
cd exam-1/{your-github-username}
pnpm install
pnpm dev
```

### 3. PR 제출

```bash
# GitHub에서 main 브랜치로 PR 생성
```

## 📝 PR 컨벤션

- **PR 제목**: `[exam-1] OOO(이름) 과제 제출 합니다.`
- **브랜치명**: `exam-1/{github-username}`
- **base 브랜치**: `main`
- PR 본문에 **설계 판단 근거** 및 고민 헀던 부분, 함께 이야기 하고 싶은 부분들을 포함해주세요 (각 exam README 참고)

## 🤝 코드 리뷰 규칙

1. **모든 참여자**가 다른 참여자의 PR에 최소 1개 이상 리뷰를 남깁니다.
2. 리뷰는 "왜 이렇게 했는지" 에 집중합니다. 단순 스타일 지적보다 설계 판단에 대한 토론을 우선합니다.
3. 리뷰 코멘트에는 가능하면 **대안 제시**를 포함해주시면 좋을 것 같습니다.
4. 리뷰 코멘트에서는 구성원들의 발전과 성장에 도움이 될 이야기 (레퍼런스, 블로그 글 등) 도 포함해주시면 감사하겠습니다.
